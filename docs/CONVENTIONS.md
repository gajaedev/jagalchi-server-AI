# 개발 컨벤션

본 문서는 Jagalchi 프로젝트의 지속 가능한 개발을 위한 기본 원칙을 정리한다.

## 핵심 원칙
- YAGNI: 당장 필요하지 않은 복잡한 구현을 피한다.
- Clean Architecture: 도메인/서비스/인프라를 명확히 분리한다.
- 변경 용이성: 작은 모듈과 명확한 책임을 우선한다.
- 데이터 무결성: 상태 변경은 이벤트 로그/스냅샷으로 추적 가능해야 한다.

## 코드 규칙
- 함수는 단일 책임을 유지하고 200라인 이하를 지향한다.
- 입력/출력 스키마는 명시적이며 JSON 형태로 고정한다.
- 캐시 키/버전/해시가 포함되어 재현 가능해야 한다.

## 테스트 규칙
- 핵심 로직(루브릭, 랭킹, 캐시, 신뢰 점수)은 단위 테스트가 필수다.
- 실패 케이스(빈 데이터, 낮은 신뢰도 등)를 반드시 포함한다.
